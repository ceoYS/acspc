# Pain Points 분석 및 V1 범위 결정

- 작성일: 2026-04-21
- 재료: docs/interviews/01_*, docs/interviews/02_*
- 성격: 해석 문서. 인터뷰 원문은 interviews/ 에 별도 보존.

## 핵심 판단

인터뷰 01 섹션 4의 의심("우리 앱을 쓴다고 해서 실제로 pain 이 해결되는가?")
은 정당한 의심이며, 이 의심이 V1 범위 결정의 기준점이다.

사용자가 지적한 대로, 우리 앱이 해결할 수 없는 것과 해결할 수 있는 것을
분명히 구분해야 한다. 구분을 애매하게 하면 "좋은 앱인데 안 쓴다" 가 된다.

## 우리 앱이 해결 못 하는 것 (명시)

1. **현장 확인 방문 자체** — 사진 한 장으로 현장 맥락이 100% 전달 안 되는
   물리적 한계. Fieldwire·Procore도 못 풂.
2. **작업자의 지시 재확인 전화** — 작업자 앱 설치 강제 불가. 나이/습관 문제.
3. **업체의 완료 회신** — 여러 업체가 바빠서 안 찍는다는 구조적 문제.
4. **지시의 완벽한 일방향 전달** — "말로 아무리 표현해도 사진이 제일
   정확하다" 는 한계는 우리가 못 넘음.

## 우리 앱이 실제로 해결하는 것 (명시)

1. **수행팀 내부의 사진 분류 노동 (야근 1.5~3시간)** — 이건 단독 완결 가능.
   작업자 참여 불필요. 업체 회신 불필요.
2. **사진대지 엑셀 양식화 수작업** — 양식 1종 고정 자동 생성.
3. **업체별 폴더 분배 수동 작업** — 업체 태그 기반 자동 분류 (엑셀 자동 생성으로 대체, V2 이월 항목).
4. **수행팀 내부의 follow-up 기억 의존** — 누구한테 무엇을 시켰는지 기록 영속화.
   (단, 이건 수행팀 내부 담당자 추적. 작업자 추적 아님.)
5. **동삼보드판의 "분류 기능 부재"** — 촬영 시점에 태그가 붙음.

## 인터뷰 01 의문에 대한 정면 답변

> "우리 앱을 쓴다고 해서 달라지는 점은 (a) follow-up 가능, (b) 실시간 조회
> 인데, 이 점이 pain point와 상관이 있을지 의문."

답: **이 두 점은 핵심 가치가 아니다.** 핵심 가치는 다른 곳에 있다.

V1 의 진짜 핵심 가치는 "퇴근 후 1.5~3시간의 야근 노동이 0분이 된다" 이다.
이건 인터뷰 02 섹션 2, 4, 5에서 드러난 반복된 페인이다. follow-up 과
실시간 조회는 부수 효과이지 주된 가치 제안이 아니다.

즉 V1 을 팔 때 "협업이 편해집니다" 가 아니라 "야근이 사라집니다" 로
팔아야 한다.

## 제품 원칙 (PP 에서 도출)

이 원칙들은 인터뷰 원문에서 직접 도출된 것이며, V1 의사결정에서 타협
대상이 아니다.

1. **작업자는 V1 에서 "사용자" 가 아닌 "콘텐츠 대상"** — PP-05, PP-06 근거
2. **업체 회신을 기대하지 않는다. 가치는 수행팀 편의에서 완결된다.** — PP-04 근거
3. **사진이 곧 위치 정보** — PP-10 근거. 도면 핀 기능은 V1 제외
4. **성공의 증명은 "야근 시간 감소"** — PP-01 근거. follow-up 은 부수 효과
5. **Lock-in 경로는 작업자가 아니라 수행팀·본사 쪽** — 인터뷰 01-§2 근거.
   작업자 참여는 2~3년 뒤 일

## 표본 확장 메모 (2026-05-11, 김은수 인터뷰 03 후)

원문: docs/interviews/03_김은수_분류축_재확인.md
배경: 김민성 (인터뷰 01/02) 외 두 번째 도메인 사용자 표본 확보.

### 기존 PP 재확인 사항

- **PP-01 (분류 야근)**: 재확인. 김은수 발언 "사진대지 엑셀로 만들 때
  사진 하나하나 찾아 넣기가 귀찮아서 생각났던 거였어" — V1 코어 가치
  검증.
- **PP-09 (자동 분류 부재)**: 재확인. 김민성의 동삼보드판 자동 분류
  부재와 동일.
- **PP-10 (사진이 곧 위치)**: 재확인. 김은수 "위치도 따로 사진에
  그려주거나 설명을 덧붙여야 한다" — 위치 정보 부담은 두 표본 공통.

### 신규 발견 (표본 확장으로 드러난 것)

다음 4개는 김은수 인터뷰에서 새로 드러난 사항. V1 변경 검토 대상 1개
(N-2) + V2 우선순위 자료 3개 (N-1, N-3, N-4).

- **N-1. 감리 점검 트리거** — 감리 점검 직전에는 엑셀 장표 정리가
  강제. 평상시는 엑셀 외 형식 가능. V1 영향: 출력 trigger 가 "엑셀
  생성" 단일 행동으로 가정된 점은 유지 가능하나, 평상시 출력 채널은
  V2 다양화 후보.

- **N-2. 엑셀 출력 시 사진 시간순 정렬** — 김은수 "사진대지에 사진을
  시간순으로 다 넣어야 하거든". V1 영향: docs/01 §6 (엑셀 출력 스펙)
  에 정렬 축이 명시 안 됨. **V1 보강 후보** — docs/01 §6 에 "시간
  오름차순 정렬" 1줄 추가 검토. (별도 chunk 결정.)

- **N-3. 평상시 분류 축 = 위치 > 공종** — 김은수 "위치별로 주로
  하고, 공종별로도 정리해서 사진대지를 제공". V1 영향: V1 출력은
  업체별 필터 1축만 (docs/01 §2 #5). 정렬 축 사용자 선택은 V2 후보.

- **N-4. 출력 채널 다양화 (HTML 포스터 / 앱 내 뷰)** — 사용자 의도:
  근로자에게는 엑셀 부담 큼. HTML 포스터 또는 앱 내 정리 뷰가 더
  적합. V2 후보 (현 docs/01 §4 "엑셀 양식 다양화 V2+" 의 하위 범주로
  편입).

### 기각 후보 (acspc 영역 밖)

- **HPMS 보충작업사전승인 연동**: 김은수가 직접 언급. 다만 HPMS는
  외부 의무 시스템. acspc 본 영역 (사진 → 위치/공종 분류 → 출력)
  밖. 별도 제품 후보로 보류.

### 제품 원칙 영향

- 원칙 1 (작업자는 V1 사용자 아님): 재확인 — 김은수도 "근로자들에게
  알려줄 때에는 사진만 보내면 돼" 라고 동일 패턴 (작업자 앱 강제 안 함).
- 원칙 4 (성공 = 야근 감소): 재확인 — "엑셀 만들 때 귀찮아서 생각났던
  거였어" 가 동일 동기.
- 새 원칙 후보 없음. 기존 5개 원칙 유지.

### V2 우선순위 회의 시 사용 자료

- N-2 (시간순) → V1 docs/01 §6 보강 후보로 분리됨
- N-3 (정렬 축 다양화), N-4 (출력 채널 다양화) → V2 핵심 후보 2종
- 김민성 (인터뷰 01/02) + 김은수 (인터뷰 03) 합산 결과 표본 확장 의의
