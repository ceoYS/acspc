## 1. 제품 기능 요약

원청사 수행팀 매니저가 촬영 전에 위치·공종·내용·업체를 드롭다운으로
선택한다. 태그 유지 상태로 연속 촬영하면 갤러리 앨범에 저장되고
업체별 사진대지 엑셀이 자동 생성된다.

일일 노가다성 엑셀 작업 시간 목표: 1.5~3시간 → 0~30분.

## 2. V1 이 하는 일 (6개)

1. **프로젝트 단위 마스터 설정** — 위치(동/호/구역), 공종, 업체 목록을
   촬영 전 등록. 프로젝트 생성 시 일괄 또는 개별 추가.
2. **촬영 전 태그 선택** — 위치·공종·내용·업체를 드롭다운으로 선택한
   상태에서 카메라 진입.
3. **연속 촬영** — 선택된 태그가 유지된 채로 여러 장 연속 촬영.
4. **갤러리 동기화** — 촬영 사진을 폰 갤러리의 사용자 지정 앨범에 저장.
   iOS 제약으로 앨범 구조는 평면 (중첩 폴더 미지원). V1 은 평면 앨범
   1단계만 사용.
5. **업체별 사진대지 엑셀 자동 생성** — 태그 기반 필터링으로 엑셀 1개 출력.
6. **동기화 수동/자동 트리거** — 서버 업로드 제어.

### 2.1 연속 촬영 모드 동작

- 같은 태그 조합으로 셔터를 여러 번 연속 누름. 태그는 유지.
- 중간에 내용(content_text)만 바꾸려면 화면 하단 내용 필드 탭해서 수정.
- 위치·공종·업체는 카메라 화면에서 변경 불가. 나가서 재선택해야 함.
- **취소/나가기 시**: 지금까지 찍은 N장을 저장할지 확인 팝업.
- 중단해도 이미 저장된 사진은 보존.

## 3. 데이터 스키마 (최소)

- **Project**: id, 프로젝트명(=공종명), 생성일
- **Location** (위치 태그): id, project_id, 이름 (예: "101동_503호_주방")
- **Trade** (공종 세부): id, project_id, 이름
- **Vendor** (업체): id, project_id, 이름
- **Photo**: id, project_id, location_id, trade_id, vendor_id,
  content_text, 촬영일시, 이미지 경로, 갤러리 경로

주:
- Location / Trade / Vendor 는 **프로젝트 단위 마스터**, 드롭다운 선택.
  새 값은 드롭다운 "+ 추가" 로 즉시 등록.
- content_text 는 컬럼(자유 입력) + 최근 10개 자동완성 UX.
- **Location 태그 ≠ 갤러리 앨범**. 기본값은 Location 이름이 자동으로
  앨범 이름이 되며, 사용자가 앨범 생성 시 다른 이름으로 지정 가능.
- 앨범은 평면 구조 (중첩 폴더 미지원). 구분이 필요하면 앨범 이름 자체에
  "101동_503호_주방" 처럼 구분자를 포함.
- EXIF GPS 는 저장하지 않음.

## 4. V1 비포함 (명시적 제외) {#v1-exclusions}

각 제외 항목은 docs/02_pain_points_analysis.md PP 테이블에 근거.

### 원칙상 제외
- 작업자용 앱 / 기능 (작업자는 V1 사용자 아님)
- 업체 회신 / 승인 플로우 (회신 기대 안 함)
- 도면 핀 기능 (사진이 곧 위치 정보)

### UX 속도 우선으로 제외
- AI 자동 공종 분류 (2초 입력 UX 가 우선)
- 음성 인식 태그 입력 (V2)

### V2+ 범위
- 공정표 / 절대공정
- 엑셀 양식 다양화
- 도면 위에 사진 배치 뷰
- 카톡 연동
- 실시간 협업 / 조회
- 멀티 기기 동기화 충돌 해결 (V1 은 단일 기기 가정)

## 5. Success Metrics

### 5.1 주지표
- 지표: 주간 사진대지 엑셀 생성 건수 (Design Partner 3곳 평균)
- 현재: 0건/주
- 목표: 3건/주 이상
- 측정 주체: 앱 내부 카운터 (엑셀 생성 완료 이벤트)
- 측정 기간: 온보딩 후 8주

### 5.2 보조지표 (기술적으로 측정 가능한 경우 기록)
- 지표: 엑셀 1개 생성 평균 소요 시간 (분)
- 측정 방법: 첫 사진 촬영 시점 → 엑셀 생성 완료 시점 사이 경과 시간
- 목표: 10분 이하 (200장 기준)

### 5.3 기술 제약 (주지표 달성 전제 조건)
- 사진 1장당 태그 입력 시간 ≤ 2초 (연속 촬영 모드)
- 200장 촬영 종료 후 엑셀 생성 ≤ 10분
- 하루 300장 촬영 시 앱 크래시 0회

### 5.4 사용 빈도 기준선
Design Partner 온보딩 후 주 3회 이상 엑셀 생성이 자연 발생해야 타깃
이 맞는 것으로 해석. 8주간 주 3회 미달 시 타깃 재검토.

## 6. 엑셀 출력 스펙

### 6.1 레이아웃 (240223_사진대지.xlsx 기준 고정)

- 시트당 사진 2장
- 시트명: "1", "2", "3", …
- 제목: C1:H1 병합 "사   진   대   지"
- 프로젝트명: A3:J3 병합
- 사진 1: 행 5, B5:I5 병합, 높이 275.1
- 사진 1 메타: 행 7~8 (공종 / 위치 / 내용 / 일자)
- 사진 2: 행 11, B11:I11 병합
- 사진 2 메타: 행 13~14
- 폰트 / 컬럼 폭 / 행 높이 원본 준수

### 6.1.1 정렬 순서 (감리 제출 강제 요건)

- 엑셀 출력 시 사진은 **created_at 오름차순 정렬** (시간순).
- 근거: docs/interviews/03 김은수 발언 "사진대지에 사진을 시간순으로
  다 넣어야 하거든". 감리 점검 직전 제출 시 강제 요건.
- 평상시 출력 (위치별/공종별 정렬) 은 V2 후보 (PP §표본 확장 메모 N-3).

### 6.2 파일명 규칙

- 기간 지시 (복수 날짜): `[공종명]_[업체명]_[YYMMDD-YYMMDD].xlsx`
- 단일일: `[공종명]_[업체명]_[YYMMDD].xlsx`
- 특수문자 (`/ \ : * ? " < > |`) 는 `_` 로 치환
- V1 에서는 프로젝트명 = 공종명 (1:1 매칭)
- **재생성 시 접미 규칙**: 동일 공종·업체·기간으로 재생성 시 파일명
  끝에 `(1)`, `(2)` … 순차 접미.
  예: `외벽타일_대성건설_260420-260426(1).xlsx`

### 6.3 파일 구조

- 출력은 엑셀 1개 (zip 없음)
- 사진은 원본 해상도로 셀 안에 embed (참조 링크 X)
- embed 후에도 사람이 수동 편집 가능한 구조 유지
- 업체가 여러 개면 업체별로 엑셀이 개별 생성됨

## 7. 비기능 요구

- 오프라인 촬영 가능 (네트워크 없이 기록). 단 엑셀 생성은 온라인 필수.
- 엑셀 생성은 서버 측 (Supabase Edge Function) 에서 실행. 오프라인
  상태에서는 사진 촬영·태그 저장까지 가능, 엑셀 생성은 재연결 후.
- 서버 업로드: **수동 버튼 트리거가 기본**, 설정에서 "Wi-Fi 시 자동" 옵션
  켤 수 있음. 기본값은 수동.
- 동기화 완료된 사진은 앱 내부 DB 에서 제거, 갤러리에만 유지
  (로컬 저장 용량 관리)
- 사진 원본 해상도 보존 (갤러리 저장 시에도)
- 하루 300장 처리 시 앱 멈춤 없음
- 개인정보 최소 수집 (엔터프라이즈 SSO V1 불필요)
- V1 은 단일 기기 사용 가정 (멀티 기기 동기화 충돌 V2+)

## 8. 작업자 접점에 대한 제품 원칙

- 작업자는 V1 에서 "사용자" 가 아닌 "콘텐츠 대상"
- 작업자 앱 사용 가정 기능은 V1 전면 제외
- 수행팀 단독 사용으로 가치가 닫히는 기능만 V1 포함
- Lock-in 경로는 작업자가 아니라 수행팀·본사 쪽
