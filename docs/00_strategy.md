## 1. 왜 만드는가 (Problem)

한국 건설 현장 수행팀 매니저(막내~대리급)는 하루 100~300장의 하자 사진을
찍는다. 입주 전 사전점검 시기에는 하루 300장까지 올라간다. 현재 워크플로는
이렇다.

1. 현장에서 사진 촬영 (동삼보드판 등 유틸리티 앱 사용)
2. 퇴근 후 (17:30~) 사무실에서 사진을 엑셀 사진대지 양식에 수작업으로 정리
3. 업체별로 폴더 분류해 카톡방에 뿌림

이 시간은 가치 창출이 아니라 노동 낭비다. 한 사람당 월 30~60시간,
연 360~720시간이 노가다성 엑셀 작업 (사진 분류 + 엑셀 양식화 +
업체별 폴더 분배) 에 사용된다.

## 2. 해결 목표

수행팀 매니저 1인의 일일 노가다성 엑셀 작업 시간을 1.5~3시간에서
0~30분으로 단축한다.

### 2.1 측정 방법
- 주간 사진대지 엑셀 생성 건수 (목표: 0 → 3 이상)
- 엑셀 1개 생성 평균 소요 시간 (목표: 측정 가능 시 기록)

### 2.2 범위 외 부수 효과
협업 기능, 실시간 조회, follow-up 은 V1 범위에서 부수 효과로만 취급.
우선순위 판단 기준은 2.1 측정 지표 개선 여부.

## 3. 누구를 위한 제품인가 (Target)

- **1차 사용자**: 원청사(종합건설사) 수행팀 매니저 (막내~대리급)
- **구매 결정자 가설**:
  - 개인 구독: 본인 결제 월 1~2만 원 수준 (검증 필요)
  - 팀 구독: 현장 단위 월 정액 (확장 경로)
- **명시적으로 타깃이 아닌 사람**: 협력업체 작업자, 협력업체 사무실, 본사 임원

## 4. 왜 지금 시장이 열려있는가 (Why Now)

- 국내 콘테크 시장은 콘업·카스웍스가 각각 감리·안전에 몰입해 있음.
  "사진 → 사진대지 → 업체 전달" 파이프라인 자동화는 비어 있음.
- Fieldwire, Procore 는 글로벌 협업 플랫폼이라 한국형 사진대지 양식·
  업체 카톡 전달 워크플로를 커버하지 않음. 앞으로도 커버 안 할 가능성
  높음 (ROI 문제).
- 중대재해처벌법 이후 증빙 수요가 구조적으로 증가 중.

## 5. 경쟁 구도에서의 위치 (Positioning)

이 제품은 건설 협업 플랫폼이 아니다. 사진 기반 문서 자동화 SaaS 다.
카테고리가 다르다.

- vs 동삼보드판: 동삼보드판은 "보드판 1장 만들기" 유틸리티. 우리는
  "하루치 사진 일괄 → 업체별 엑셀" 파이프라인.
- vs 콘업 / 카스웍스: 그들은 협업 플랫폼. 작업자 참여 전제. 우리는
  수행팀 단독 사용.
- vs Fieldwire / Procore: 그들은 양방향 협업. 학습 곡선 길고 가격
  비쌈. 우리는 단방향 출력 파이프라인.

## 6. 차별화 축 (Moat)

1. **측정 가능한 단일 KPI**: 주간 엑셀 생성 건수. 경쟁사는 "생산성
   향상" 등 다축 지표 사용.
2. **입력 속도**: 사진 1장당 태그 입력 ≤ 2초 (경쟁사 측정 평균
   10~30초 추정).
3. **수행팀 단독 완결**: 협력사 앱 설치 0건 필요. 업체 온보딩 시간
   0분.
4. **엑셀 양식 1종 고정**: 사용자 제공 240223_사진대지.xlsx 기반.
   V1 학습 시간 0분 (기존 서식 그대로).
5. **오프라인 촬영**: 네트워크 연결 0 상태에서 촬영·태그 저장 가능.
   재연결 시 수동 버튼 또는 Wi-Fi 자동 업로드.

## 7. 장기 확장 경로 (Roadmap Axes)

현재 V1 은 건설 마감 하자 사진대지 자동화에 집중하지만, 엔진
(사진 + 태그 + 양식화 + 업체별 분배)은 다음으로 확장 가능하다.

- P1: 입주 전 사전점검 (하루 200~300장 구간. 같은 엔진 재사용)
- P1: 소음 측정 민원 대응 양식 (양식만 바꾸면 됨)
- P2: 제조업 현장 점검
- P2: 시설물 진단 리포트
- V2: 음성 인식 태그 입력 (손 안 대고 말로만 분류. 현장에서 장갑 낀 채
  사용 가능)
- V2+: 공정 생산성 DB / AI 기반 일정 검증 (수행팀 실적 데이터가 쌓인 뒤)

## 8. Success Metrics

### 8.1 주지표
- 지표명: 주간 사진대지 엑셀 생성 건수 (Design Partner 3곳 평균)
- 현재: 0건/주 (앱 도입 전)
- 목표: 3건/주 이상
- 측정 기간: Design Partner 온보딩 후 8주

### 8.2 보조지표
- 지표명: 엑셀 1개 생성 평균 소요 시간 (앱 내부 측정)
- 현재: 측정 불가 (수동 작업)
- 목표: 10분 이하 (200장 촬영분 기준)
- 측정 기간: 주지표와 동일

### 8.3 실패 판정
주지표 목표를 8주 내 달성 못 하면 타깃 / 가치 제안 재검토.
