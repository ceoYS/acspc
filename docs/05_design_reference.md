**제목**: 05. Design Reference
**한 줄 요약**: acspc V1 디자인 규칙 문서. 벤치마크: CompanyCam (모바일), Airtable (데스크탑), 토스 (인터랙션).

# 1. 벤치마크 차용 원칙
## 1.1 CompanyCam
**차용 O**
- 카메라 진입 전 태그를 고정하고 촬영 중 동일 맥락을 유지하는 패턴을 차용한다. 적용: 카메라 진입 직전 태그 확정 UI, V1-2, V1-3.
- 태그 기반 사진 필터링과 결과물 생성의 연결 패턴을 차용한다. 적용: 사진 목록 필터, 업체별 엑셀 생성 트리거, V1-5.
- 현장 오프라인 촬영 후 사무실 또는 현장에서 보고서를 만드는 흐름을 차용한다. 적용: 동기화 상태 피드백, 수동/자동 업로드 제어, V1-6.
**차용 X**
- 팀 코멘트, 멘션, 협업 대화 중심 구조는 차용하지 않는다. acspc V1 은 수행팀 단독 완결이 원칙이며 작업자 참여와 실시간 협업은 범위 외다.
- AI 자동 요약, 음성 기반 자동 문서화 중심 경험은 차용하지 않는다. acspc V1 의 우선순위는 2초 입력과 고정 서식 엑셀 생성이지 지능형 보조가 아니다.
## 1.2 Airtable
**차용 O**
- 좌측 뷰 전환, 중앙 레코드 목록, 우측 상세 프리뷰로 이어지는 정보 구조를 차용한다. 적용: 데스크탑 갤러리 검토, V1-4, V1-5.
- 필터, 그룹, 정렬을 얕은 단계에서 빠르게 조합하는 패턴을 차용한다. 적용: 업체별 사진 필터링, 프로젝트 단위 검토, V1-1, V1-5.
- 레코드 기반 출력 레이아웃 사고방식을 차용한다. 적용: 사진 레코드 검토 후 엑셀 생성 트리거, 결과물 직전 확인 화면, V1-5.
**차용 X**
- 범용 데이터베이스 편집 자유도는 차용하지 않는다. acspc 는 자유 모델링 도구가 아니라 사진 기반 문서 자동화 SaaS 이다.
- 복잡한 다중 뷰 설정과 사용자별 퍼스널 뷰 개념은 차용하지 않는다. V1 은 현장 매니저 1인의 빠른 분류와 출력이 목표이며 학습 비용을 늘리지 않는다.
## 1.3 토스
**차용 O**
- 짧고 단정한 문구, 즉시 이해되는 상태 피드백을 차용한다. 적용: 토스트, 오류 문구, 업로드 상태, V1-6.
- 큰 터치 영역과 단계 축소 중심의 마이크로인터랙션을 차용한다. 적용: 하단 캡처 바, 태그 선택 바텀시트, V1-2, V1-3.
**차용 X**
- 금융 서비스 특유의 과도한 요약 카드 구조는 차용하지 않는다. acspc 의 핵심 객체는 돈이 아니라 사진과 태그다.
- 감성적 브랜딩 애니메이션은 차용하지 않는다. 현장 환경과 저사양 기기에서는 속도 우선 원칙을 지킨다.
## 1.4 기준선 기록
- Autodesk Build: 업계 표준 수준의 사진 갤러리, 필터, 리포트 경험은 참고하되 본문 벤치마크에는 포함하지 않는다.
- 공사 보드판: 국내 기존 대안의 결과물 중심 흐름은 참고하되 분류 기능 부재를 넘어서는 기준선으로만 기록한다.

# 2. 사용자 흐름 (V1 6기능)
## V1-1 프로젝트 단위 마스터 설정 (데스크탑 + 모바일)
└─ 웹: 프로젝트 목록 진입 → 프로젝트 생성 → 위치/공종/업체 입력 → 저장
└─ 웹: 각 목록에서 기존 값 확인 → 상단 `+ 추가` → 즉시 입력 모달 → 저장 후 선택 상태 유지
└─ 모바일: 프로젝트 동기화 완료 → 프로젝트 선택 → 드롭다운에서 위치/공종/업체 불러오기
└─ 모바일: 값이 없으면 `+ 추가` → 바텀시트 입력 → 저장 → 같은 흐름으로 즉시 선택
## V1-2 촬영 전 태그 선택 (모바일)
└─ 프로젝트 선택 완료 → 카메라 진입 전 태그 선택 화면 진입
└─ 위치 선택 → 공종 선택 → 내용 입력 또는 최근값 선택 → 업체 선택
└─ 상단 태그 chip 4개 확인 → 셔터 사용 가능 상태 활성화 → 카메라 화면 진입
## V1-3 연속 촬영 (모바일)
└─ 카메라 화면 진입 → 상단에 위치/공종/업체 chip 고정 표시
└─ 하단 고정 내용 필드 확인 → 필요 시 탭 → 키보드 입력 → 확인 → 프리뷰 복귀
└─ 셔터 탭 → 로컬 저장 → 즉시 다음 셔터 가능 상태 복귀
└─ 같은 태그 조합으로 반복 촬영
└─ 나가기 탭 → `지금까지 찍은 N장을 저장할까요?` 확인 → 저장 또는 폐기
## V1-4 갤러리 동기화 (모바일 + 데스크탑 확인)
└─ 모바일: 촬영 직후 로컬 저장 → 사용자 지정 앨범에 저장
└─ 모바일: 온라인이면 업로드 큐 전송 → 오프라인이면 로컬 큐 유지
└─ 모바일: 업로드 시점에만 `동기화 대기 중 (N장)` 안내 → 3초 후 자동 해제
└─ 웹: 동기화 완료 사진이 프로젝트 갤러리에 반영 → 썸네일/태그/촬영일시 확인
## V1-5 업체별 사진대지 엑셀 자동 생성 (데스크탑)
└─ 웹: 프로젝트 진입 → 좌측 뷰 선택 또는 기본 갤러리 열기
└─ 웹: 상단 필터에서 업체 선택 → 필요 시 위치/공종 보조 필터 적용
└─ 웹: 중앙 사진 레코드 검토 → 우측 프리뷰에서 선택 개수와 생성 범위 확인
└─ 웹: `엑셀 생성` 탭 → 서버 생성 진행률 확인 → 완료 후 다운로드 또는 공유
## V1-6 동기화 수동/자동 트리거 (모바일 + 데스크탑)
└─ 모바일: 자동 업로드 기본값 유지 → 네트워크 가능 시 백그라운드 업로드 시도
└─ 모바일: 설정 화면 진입 → 대기 큐 개수 확인 → `지금 동기화` 탭
└─ 모바일: 오프라인 상태에서 업로드 시도 시 안내 문구 노출 → 촬영 기능은 계속 사용 가능
└─ 웹: 업로드 완료 후 최신 사진 반영 → 재진입 없이 목록 갱신

# 3. 비주얼 스타일 가이드
## 3.1 색상 팔레트
**Primary (주요 액션)**
- 값: `bg-slate-900 text-slate-50`
- 사용처: 확인 버튼, 상단 헤더, 주요 저장 액션
**Accent (촬영 전용, 다른 용도 금지)**
- 값: `bg-orange-600 text-slate-50`
- 사용처: 셔터 버튼, 촬영 가능 상태 인디케이터
- 사유: 건설 현장 안전 컬러 계열, 햇빛 가독성 유지
**Surface**
- Surface: `bg-white`
- Surface-elevated: `bg-slate-50`
**Semantic**
- Success: `bg-emerald-700`
- Danger: `bg-red-700`
- Warning: `bg-amber-700`
- Info: `bg-blue-700`
**Border**
- 기본: `border-slate-300`
- 강조: `border-slate-700`
- 사용 최소화, 간격과 배경 대비를 우선한다.
**Text**
- Primary text: `text-slate-900`
- Secondary text: `text-slate-700`
- Disabled text: `text-slate-400`
**현장 가독성 규칙**
- 명도 대비 최소 7:1 을 유지한다.
- 순수 회색 단독 사용을 금지한다.
- 햇빛 반사와 저가 패널 환경에서 회색 계열 단독 대비는 급격히 무너진다.
## 3.2 타이포그래피
**폰트**
- iOS: `system-ui`
- Android: `Roboto`
- Web: `system-ui, sans-serif`
- 커스텀 폰트 로드를 금지한다. 회사망 차단 이력과 Google Fonts 비의존 원칙을 유지한다.
**크기 스케일**
- Display: `text-2xl` `font-bold`
- Title: `text-xl` `font-semibold`
- Body: `text-base` — 16pt 이하로 낮추지 않는다.
- Label: `text-sm` `font-medium`
- Caption: `text-xs text-slate-500`
**굵기**
- `font-normal` 과 `font-semibold` 2단계만 사용한다.
- `font-bold` 는 Display 에만 사용한다.
**Line-height**
- Body: `leading-normal`
- Title: `leading-snug`
## 3.3 공간 규칙
**그리드**
- 4pt 기반을 사용한다.
**터치 타겟**
- 최소: `min-h-12 min-w-12`
- Primary button: `min-h-14`
- Capture Bar 셔터: `size-16`
**패딩/간격**
- 컨테이너: `p-4`
- 섹션 간격: `gap-6`
- 컴포넌트 간: `gap-3`
**모서리**
- 기본 카드/버튼: `rounded-lg`
- Primary button: `rounded-xl`
- Chip/Tag: `rounded-full`
**모바일 safe area**
- 하단 캡처 바는 `min-h-20` 과 `safe-area-inset-bottom` 을 항상 예약한다.

# 4. UI 컴포넌트 가이드
## 4.1 Button (Primary / Secondary / Destructive)
**용도**
- Primary 는 저장, 확인, 생성에 사용한다.
- Secondary 는 닫기, 취소, 보조 이동에 사용한다.
- Destructive 는 삭제, 폐기에만 사용한다.
**Tailwind 클래스**
- Primary: `min-h-14 rounded-xl bg-slate-900 px-4 text-slate-50`
- Secondary: `min-h-12 rounded-lg bg-slate-100 px-4 text-slate-900`
- Destructive: `min-h-12 rounded-lg bg-red-700 px-4 text-white`
**상태**
- default: 기본 클래스 유지
- hover: 데스크탑만 `opacity-90`
- active: `opacity-80`
- disabled: `bg-slate-200 text-slate-400`
**금지사항**
- 아이콘만 있는 Primary 버튼을 만들지 않는다.
- 두 개 이상의 Primary 버튼을 한 화면 하단에 병치하지 않는다.
## 4.2 Capture Bar (모바일 전용, 하단 고정)
**용도**
- 한 손 조작 영역에서 촬영, 내용 수정, 나가기 액션을 제공한다.
**Tailwind 클래스**
- 래퍼: `fixed inset-x-0 bottom-0 min-h-20 bg-white px-4 pb-safe pt-3`
- 셔터: `size-16 rounded-full bg-orange-600`
- 보조 버튼: `min-h-12 min-w-12 rounded-full bg-slate-100`
**상태**
- default: 셔터 활성
- active: `opacity-80`
- disabled: `bg-slate-300`
- offline: 시각 상태를 바꾸지 않고 배너로만 알린다
**금지사항**
- 화면 중앙 이상으로 올려 배치하지 않는다.
- 스와이프 제스처를 주요 촬영 액션에 연결하지 않는다.
## 4.3 Tag Chip
**용도**
- 위치, 공종, 업체, 내용 상태를 짧게 요약하고 현재 선택값을 유지한다.
**Tailwind 클래스**
- 미선택: `min-h-12 rounded-full bg-slate-100 px-3 text-slate-700`
- 선택: `min-h-12 rounded-full bg-slate-900 px-3 text-slate-50`
- 강조 보더 필요 시: `border border-slate-700`
**상태**
- default: 미선택 또는 선택 상태 표시
- hover: 데스크탑만 `opacity-90`
- active: `opacity-80`
- disabled: `bg-slate-100 text-slate-400`
- 연속 촬영 중: 기존 선택을 유지하고 내용만 수정 가능
**금지사항**
- 카메라 화면 안에서 위치·공종·업체 chip 을 수정 가능하게 만들지 않는다.
- 한 줄에 4개 이상 강제 배치하지 않는다.
## 4.4 Card (사진 썸네일)
**용도**
- 사진 미리보기, 태그 요약, 선택 상태 표시, 상세 프리뷰 진입에 사용한다.
**Tailwind 클래스**
- 공통: `rounded-lg bg-white p-3`
- 썸네일: `aspect-square` 또는 `aspect-[4/3]`
- 선택 상태: `border border-slate-700`
- 모바일 리스트: `flex gap-3`
- 데스크탑 그리드: `grid gap-3`
**상태**
- default: 썸네일 + 메타데이터 노출
- hover: 데스크탑만 `bg-slate-50`
- active: `opacity-90`
- selected: 강조 보더 + 체크 노출
- disabled: 선택 불가, `opacity-50`
**금지사항**
- 사진 위에 긴 텍스트를 중첩하지 않는다.
- 그림자만으로 계층을 구분하지 않는다.
## 4.5 Input (텍스트 / 검색)
**용도**
- 데스크탑 검색, 프로젝트 마스터 입력, 내용 자동완성 입력에 사용한다.
**Tailwind 클래스**
- 입력창: `min-h-12 rounded-lg border border-slate-300 bg-white px-3 text-slate-900`
- 포커스: `border-slate-700`
- 검색 입력: `min-h-12 rounded-lg bg-slate-50 px-3 text-slate-900`
**상태**
- default: 일반 입력
- hover: 데스크탑만 `border-slate-700`
- active: `border-slate-700`
- disabled: `bg-slate-100 text-slate-400`
- error: `border-red-700 text-slate-900`
**금지사항**
- 모바일에서 긴 선택 리스트를 인라인 드롭다운으로 열지 않는다.
- placeholder 만으로 라벨을 대체하지 않는다.
## 4.6 Bottom Sheet (모바일)
**용도**
- 드롭다운 대체, 최근값 선택, `+ 추가` 입력 진입에 사용한다.
**Tailwind 클래스**
- 래퍼: `rounded-t-xl bg-white p-4`
- 리스트 항목: `min-h-12 rounded-lg px-3 text-slate-900`
- 최근 5개 영역: `gap-2 border-b border-slate-300 pb-3`
**상태**
- default: 최근 5개 상단 고정, 전체 목록 아래 배치
- active: 선택 항목 `bg-slate-900 text-slate-50`
- disabled: `text-slate-400`
- search: 항목 수가 스크롤 시점을 넘을 때만 검색 입력 표시
**금지사항**
- 전체 화면 모달로 강제 전환하지 않는다.
- 최근값 영역을 숨기지 않는다.

# 5. 인터랙션 및 애니메이션 지침
## 5.1 인터랙션 원칙
- 탭을 우선한다.
- 스와이프, 핀치, 드래그를 주 액션에 사용하지 않는다.
- 롱프레스는 보조 액션에만 사용한다.
- 모바일 햅틱은 성공 `light`, 에러 `medium` 2단계만 사용한다.
- 데스크탑 단축키는 4개로 고정한다: `Cmd+K` 명령 팔레트, `/` 검색 포커스, `J` 다음 사진, `K` 이전 사진.
## 5.2 애니메이션
- 모든 전환은 150ms 이하로 제한한다.
- 페이지 전환은 `fade 100ms` 를 사용한다.
- 버튼 피드백은 `opacity 80%` 50ms 로 처리한다.
- 과도한 blur 와 shadow 를 금지한다.
## 5.3 피드백
- 토스트 위치는 모바일 하단, 웹 우상단으로 고정한다.
- 오류는 필드 하단 빨간 텍스트와 `border-red-700` 으로 표시한다.
- 성공은 녹색 토스트를 2초 노출 후 자동 사라지게 한다.
- 오프라인 상태는 상시 배너로 두지 않고 업로드 시점에만 노출한다.
- 나가기 확인은 촬영 중단 시에만 모달로 노출한다.

# 6. 접근성 + 반응형
## 6.1 접근성 (a11y)
- 명도 대비 AAA 7:1 을 유지한다.
- 본문 폰트 크기는 최소 16pt 로 유지한다.
- 터치 타겟은 48pt 이상으로 유지한다.
- 아이콘 전용 버튼에는 `aria-label` 을 필수로 부여한다.
- 데스크탑은 Tab 순서를 시각 흐름과 동일하게 맞춘다.
- 모바일은 VoiceOver 와 TalkBack 을 모두 지원한다.
## 6.2 반응형
- 모바일 기준 폭은 375px~430px 로 설계한다.
- 태블릿 768px 는 2컬럼으로 전환한다.
- 데스크탑 1024px 이상은 3컬럼으로 전환한다.
- 데스크탑 3컬럼 구조는 좌 사이드바, 중앙 갤러리, 우 프리뷰/엑셀 생성 트리거로 고정한다.
- 브레이크포인트는 Tailwind 기본 `sm`, `md`, `lg` 만 사용한다.

# 7. acspc 특화 규칙 (8개)
1. 현장 햇빛: 순수 회색 단독 사용을 금지하고 WCAG AAA 대비를 유지한다.
2. 장갑 낀 손: 모든 주 액션은 48pt 이상 터치 타겟으로 설계하고 드래그와 핀치를 금지한다.
3. 한 손 조작: 주요 액션은 화면 하단 1/3 에 배치한다.
4. 2초 입력: 연속 촬영 시 다음 사진 진입까지 태그 유지 상태로 3탭 이내 입력을 끝낸다.
5. 데이터 소실 방지: 사진은 로컬 저장 후 서버 동기화 순서로 처리한다.
6. 애니메이션 최소: 모든 애니메이션은 150ms 이하로 제한하고 blur 와 shadow 과용을 금지한다.
7. 스와이프/핀치 금지: 현장 오동작 가능성이 큰 제스처는 주 흐름에서 제외한다.
8. 촬영 후 즉시 복귀: 셔터 후 즉시 다음 촬영 가능 상태로 복귀시킨다.
