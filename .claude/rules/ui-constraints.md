# UI Constraints — acspc

AI 가 UI 구현 시 반드시 지켜야 하는 UX 제약. docs/01 §2, §5, §7 에
분산된 규칙의 단일 참조처.

## 1. 성능 목표

### 1.1 태그 입력 속도 (docs/01 §5.3 반영)

- 드롭다운 응답: 터치 → 목록 표시 ≤ 300ms
- 전체 4개 태그 선택 총 시간: 제약 없음 (첫 사진)
- **연속 촬영 2번째 이후 사진**: 셔터 → 다음 셔터 가능 상태까지 ≤ 2초

### 1.2 기타 성능 예산 (V1 MVP 목표)

- 앱 콜드 스타트: ≤ 3초 (캐시 정상 상태)
- 카메라 화면 진입: ≤ 1.5초
- 사진 로컬 저장 (셔터 후 디스크 쓰기): ≤ 500ms
- 프로젝트 목록 로드: ≤ 1초

V1 QA 단계에서 실측. 미달 시 V1 출시 차단 사유는 아님 (V1.5 개선).

## 2. 드롭다운 패턴

- 표시: 프로젝트 내 기존 값 목록
- **"+ 추가" 버튼 위치: 목록 최상단**
- "+ 추가" 선택 시 즉시 입력 모달 → 저장 후 해당 값이 선택된 상태로
  드롭다운 닫힘
- 검색 필드는 **값 개수가 스크롤 발생 시점 초과 시** 자동 활성화 (기기
  화면 크기 의존, 보통 8~10개)
- 키보드 조작: 화살표 + Enter. 모바일은 탭만.

## 3. 연속 촬영 UX (docs/01 §2.1 반영)

- 촬영 화면 진입 전 4개 태그 (위치·공종·내용·업체) 선택 완료 상태
- 카메라 화면에서 **위치·공종·업체는 변경 불가** (나가서 재선택)
- **내용 (content_text) 만 중간 변경 가능**. 구현 방식:
  - 카메라 프리뷰 **하단 고정 필드** (프리뷰 가리지 않음)
  - 탭 시 키보드만 올라옴, 프리뷰 유지
  - 입력 후 "확인" 탭 → 프리뷰 복귀
  - 슬라이드업 또는 하단 ⅓ 오버레이 패턴
- 연속 셔터: 각 셔터 사이 ≤ 2초 입력 가능
- **취소/나가기 시 확인 팝업**: "지금까지 찍은 N장을 저장할까요?"
  - 저장: 태그 그대로 N장 DB 저장
  - 취소: N장 전부 폐기
- 저장 후 중단해도 이미 저장된 사진은 보존

## 4. 오프라인 UX

- **상시 배너 표시 안 함** (현장 오프라인은 정상 상태)
- **업로드 시점에만** 표시:
  - "네트워크 없음. 동기화 대기 중 (N장)"
  - 배너 표시 시간: **3초 후 자동 해제**. 사용자 명시 탭 시 즉시 해제.
- 동기화 대기 큐는 설정 화면에서 수동 확인 가능
- 촬영 자체는 오프라인 정상 동작

## 5. 엑셀 생성 UX

- 업체 필터 선택 → "엑셀 생성" 버튼 탭
- **온라인 필수** (docs/01 §7, tech-stack §1.6)
- 오프라인 상태에서 버튼 탭 시: "네트워크 연결 후 다시 시도" 안내
- 온라인 상태:
  - 진행률 표시 (업로드 비율 + 서버 생성 대기)
  - 완료 시 파일 다운로드 또는 공유 시트 자동 열기
- **재생성 시 파일명**: docs/01 §6.2 파일명 규칙에 `(1)`, `(2)` 접미
  (동일 공종·업체·기간 재생성). 파일명 규칙은 docs/01 §6.2 단일 원본.

## 6. 접근성 최소선

현장 사용 맥락 반영:

- **터치 영역 최소 44×44pt** (iOS HIG / Android Material 기준)
- 드롭다운·버튼은 **48×48pt 권장** (얇은 장갑 정확도 보완)
- 한 손 조작 가능한 위치 (하단 중앙)
- 텍스트 최소 크기 16pt
- 색상 대비 WCAG AA 이상 (현장 밝은 빛 가독성)
- V1 QA 단계에서 얇은 장갑 실기 테스트 수행

## 7. 금지 UI 패턴 (V1 비포함 확인)

다음은 docs/01 §4 V1 비포함 항목. UI 설계 시 제안·구현 금지.

- AI 자동 공종 분류 제안 (자동 감지 UI 금지)
- 음성 인식 버튼 / 음성 입력 필드 (V2)
- 도면 표시 / 핀 찍기 UI (V1 비포함)
- 업체 승인 / 회신 UI (V1 비포함)
- 작업자 계정 / 역할 선택 UI (V1 비포함)
- **카카오톡 SDK 직접 연동 금지** (imports, 버튼, 딥링크 생성 모두).
  OS 공유 시트 호출은 허용 (사용자가 카카오톡 선택 가능, OS 주도 동작).
- 실시간 협업 인디케이터 (V1 비포함)
- 엑셀 양식 선택 UI (V1 은 1종 고정)
